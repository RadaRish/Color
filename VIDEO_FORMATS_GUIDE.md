# Руководство по поддерживаемым форматам видео

## ✅ Поддерживаемые форматы

### Прямые ссылки на видео файлы
- **MP4** (.mp4) - Рекомендуется
- **WebM** (.webm) - Отличное качество и сжатие
- **OGG** (.ogg) - Открытый формат
- **MOV** (.mov) - Формат Apple
- **AVI** (.avi) - Старый, но поддерживается

### Data URL (Base64) - НОВОЕ!
- **data:video/mp4;base64,** - Видео, встроенное напрямую в base64
- **data:video/webm;base64,** - Поддерживаются все видео MIME-типы
- Идеально для небольших видео или офлайн-работы

### Примеры рабочих ссылок
```
https://example.com/video.mp4
https://storage.googleapis.com/sample-video.webm
https://cdn.server.com/media/clip.ogg
data:video/mp4;base64,AAAAIGZ0eXBpc29tAAA...
```

## ❌ НЕподдерживаемые форматы

### Социальные сети и платформы
- **YouTube** (youtube.com, youtu.be) - Блокируется CORS политикой
- **Instagram** (instagram.com/reel/, instagram.com/p/) - Не поддерживает прямое встраивание
- **TikTok** (tiktok.com) - Не поддерживает прямое встраивание
- **VK Video** (vkvideo.ru, vk.com/video) - Блокируется CORS политикой
- **Facebook** (facebook.com) - Ограничения безопасности
- **Twitter/X** (twitter.com, x.com) - API ограничения

## 🔧 Как получить рабочую ссылку

### Способ 1: Загрузить на хостинг
1. Загрузите видео на файловый хостинг (Google Drive, Dropbox, свой сервер)
2. Получите прямую ссылку на файл
3. Убедитесь, что ссылка заканчивается расширением (.mp4, .webm и т.д.)

### Способ 2: Конвертировать из YouTube/Instagram
1. Используйте онлайн конверторы (yt-dlp, savefrom.net и др.)
2. Скачайте видео в формате MP4
3. Загрузите на ваш хостинг
4. Используйте прямую ссылку

### Способ 3: Создать собственное видео
1. Записать видео на телефон/камеру
### Способ 4: Использовать Data URL (Base64) - НОВОЕ!
1. Конвертировать видео в Base64 (онлайн конверторы)
2. Получить строку формата `data:video/mp4;base64,AAAAIGZ...`
3. Вставить напрямую в поле URL
4. Идеально для небольших видео (до 1-2 МБ)

## ⚠️ Важные требования

### Формат ссылки
✅ **Правильно**: `https://example.com/video.mp4`
✅ **Правильно**: `data:video/mp4;base64,AAAAIGZ0eXBpc29tAAA...`
❌ **Неправильно**: `https://youtube.com/watch?v=abc123`

### CORS политика
- Хостинг должен разрешать cross-origin запросы
- Избегайте хостингов с жесткими ограничениями CORS

### Размер файлов
- Рекомендуется до 50 МБ для быстрой загрузки
- Для Data URL: до 1-2 МБ (чтобы не переполнить localStorage)
- Для больших файлов используйте сжатие

## 🛠️ Диагностика проблем

### Если видео не загружается:
1. Проверьте, что ссылка открывается в браузере
2. Убедитесь, что ссылка заканчивается расширением видео ИЛИ начинается с `data:video/`
3. Проверьте CORS настройки хостинга (не касается Data URL)
4. Для Data URL: проверьте корректность base64 строки
5. Попробуйте другой хостинг или формат

### Сообщения об ошибках:
- **"YouTube видео не поддерживается"** - Используйте прямую ссылку на MP4 или Data URL
- **"Instagram видео не поддерживается"** - Скачайте и перезагрузите на хостинг или конвертируйте в Data URL
- **"VK Video не поддерживается"** - Скачайте и перезагрузите на хостинг или конвертируйте в Data URL
- **"TikTok видео не поддерживается"** - Скачайте и перезагрузите на хостинг или конвертируйте в Data URL
- **"QuotaExceededError"** - Data URL слишком большой, используйте обычную ссылку на хостинг
- **"Неподдерживаемый формат"** - Конвертируйте в MP4/WebM
- **"Ошибка загрузки"** - Проверьте доступность ссылки

## 📋 Рекомендации

### Оптимальные настройки MP4:
- **Кодек**: H.264
- **Разрешение**: 1920x1080 или меньше
- **Частота кадров**: 30 fps
- **Битрейт**: 5000-8000 kbps

### Хостинги для видео:
- **GitHub Pages** (для небольших файлов)
- **Netlify** (бесплатный план)
- **AWS S3** (платный, но надежный)
- **Google Cloud Storage** (платный)
- **Собственный сервер** (полный контроль)
